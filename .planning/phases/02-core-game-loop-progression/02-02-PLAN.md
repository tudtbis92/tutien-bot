<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Discord Event → Redis | Untrusted user events cross Redis cooldown check |
| Redis → pg-boss | Passed events cross into job queue |
| pg-boss → DB | Job data crosses into atomic DB writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-ACT-01 | Spoofing | messageCreate userId | accept | discord.js validates user snowflakes via gateway; fabrication impossible from client side |
| T-02-ACT-02 | Tampering | Daily tu vi cap | mitigate | Atomic `UPDATE WHERE daily_tuvi + amount <= DAILY_CAP RETURNING` — race-free; concurrency:1 on ActivityWorker |
| T-02-ACT-03 | Tampering | Spam farming (messages/reactions) | mitigate | 5-layer guard: Redis NX fast-path + DB cooldown 60s + content quality gate (repeating chars, duplicate content) + daily hard cap + anomaly flag |
| T-02-ACT-04 | Tampering | Voice AFK farming | mitigate | VoiceMinuteWorker caps at VOICE_MAX_MINUTES/session; mute/deafen state checked at award time; session orphan sweep on restart |
| T-02-ACT-05 | Denial of Service | boss.send() in event handlers | accept | Fire-and-forget void — pg-boss queue absorbs load; backpressure via job expireInSeconds:120 prevents queue bloat |
| T-02-ACT-06 | Elevation of Privilege | ActivityWorker in shard.ts | mitigate | Worker registered ONLY in bot.ts (ShardingManager); shard.ts has no boss.work() calls — enforced by architecture pattern |
| T-02-ACT-07 | Information Disclosure | message.content stored in pg-boss job | mitigate | Job data expires in 120s; content used only for quality gate check, not persisted in characters table |
</threat_model>
