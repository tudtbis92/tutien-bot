    Implement `/chế_tạo` slash command:

    SlashCommandBuilder with name 'chế_tạo':
    - Integer option 'recipe_id' (required): which recipe to craft
    - String option 'tên' (optional): custom name for unique items (max 50 chars) — required if unique item rolls
    - String option 'emoji' (optional): custom emoji for unique items (max 100 chars)

    **Execute logic** (from RESEARCH.md Pattern 6):
    1. `await interaction.deferReply()`
    2. Resolve locale → t
    3. SELECT character WHERE discordId
    4. `const recipeId = interaction.options.getInteger('recipe_id', true)`
    5. START TRANSACTION: `const result = await db.transaction(async (tx) => { ... })`

    Inside transaction:
    a. SELECT recipe WHERE id = recipeId — if not found: return `{ success: false, reason: 'not_found' }`
    b. Check profession level: `getProfessionLevel(char.professionPoints, recipe.professionType) >= recipe.minProfessionLevel` — if not: return `{ success: false, reason: 'insufficient_level' }`
    c. SELECT recipe_ingredients WHERE recipeId
    d. For each ingredient: SELECT character_items WHERE characterId + itemId — if qty < required: return `{ success: false, reason: 'insufficient_materials' }`
    e. Consume ingredients (UPDATE quantity - required; DELETE WHERE quantity <= 0)
    f. Roll unique chance: `const isUnique = rollUniqueChance(getProfessionLevel(...))`
    g. If isUnique:
       - Get profession archetype from PROFESSION_UNIQUE_ARCHETYPES
       - Roll 2-4 random attributes from the archetype's attributePool with random float values
       - `const customName = interaction.options.getString('tên') ?? 'Unnamed Item'`
       - `const customEmoji = interaction.options.getString('emoji') ?? ''`
       - INSERT INTO items { nameI18nKey: archetype.uniqueItemNameI18nKey, type: inferredType, basePrice: sql\`0\`, isUnique: true, creatorCharacterId: char.id, customName, customEmoji, attributes: JSON.stringify(rolledAttributes), createdAt: sql\`now()\` } RETURNING id
       - INSERT INTO character_items { characterId: char.id, itemId: newItem.id, quantity: 1 }
       - Return `{ success: true, isUnique: true, itemNameI18nKey: archetype.uniqueItemNameI18nKey, customName, customEmoji }`
    h. If standard craft:
       - INSERT INTO character_items { characterId, itemId: recipe.resultItemId, quantity: 1 } ON CONFLICT DO UPDATE SET quantity = quantity + 1
       - Return `{ success: true, isUnique: false, itemNameI18nKey: resultItem.nameI18nKey }`

    6. Handle result outside transaction:
    - not_found → error embed `t('game:craft.recipe_not_found')`
    - insufficient_level → error embed `t('game:craft.insufficient_level', { level: recipe.minProfessionLevel })`
    - insufficient_materials → error embed `t('game:craft.insufficient_materials')`
    - success + isUnique → `buildItemEmbed({ type: 'unique_craft', ... }, t)`
    - success + !isUnique → `buildItemEmbed({ type: 'craft', ... }, t)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → Recipe lookup | recipeId from user options triggers DB read |
| Transaction → Inventory mutation | Atomic consumption of materials |
| Unique item → DB insert | User-provided custom_name/emoji written to DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-CRAFT-01 | Tampering | Partial ingredient consumption | mitigate | All ingredient checks happen BEFORE any consumption; single transaction — full commit or full rollback |
| T-02-CRAFT-02 | Tampering | Inventory quantity underflow | mitigate | character_items CHECK quantity > 0; DELETE WHERE quantity <= 0 cleans zero rows; tx.rollback on insufficient |
| T-02-CRAFT-03 | Tampering | custom_name SQL injection | mitigate | Drizzle parameterized queries — custom_name is a bind parameter, never interpolated into SQL |
| T-02-CRAFT-04 | Information Disclosure | unique item attributes exposed | accept | Item attributes stored in items.attributes JSONB; visible to item owner in /kho (inventory) — intentional |
| T-02-CRAFT-05 | Tampering | recipe_id out of bounds | mitigate | SELECT recipe WHERE id = recipeId returns null → 'not_found' error before any mutation |
| T-02-GATHER-01 | Tampering | Gathering cooldown bypass | mitigate | Redis tryAcquireCooldown 5-min per profession; server-side check, no client influence |
| T-02-GATHER-02 | Tampering | Realm requirement bypass | mitigate | char.realmId checked server-side against GATHER_REALM_REQUIREMENTS before any yield computation |
</threat_model>
