      - Determine match result from football_matches status and scores:
        - If home_score > away_score → result = 'home'
        - If home_score === away_score → result = 'draw'
        - If home_score < away_score → result = 'away'
        - Exact score result = `${home_score}-${away_score}`
      - If match status is PST/CANC/ABD/INT/SUSP (per D-07):
        - For each bet: refund wager_amount to user balance atomically, set bet status = 'void', resolved_at = NOW().

      - For normal resolution (FT/AET/PEN):
        - For each bet:
          - If bet_type = 'result' and prediction matches result → status = 'won', calculate payout, credit balance.
          - If bet_type = 'score' and prediction matches exact score → status = 'won', calculate payout, credit balance.
          - Otherwise → status = 'lost'.
        - Batch credit winners: for each won bet, UPDATE users SET balance = balance + payout.
        - Use db.transaction() for atomic resolution — all bets resolved in one transaction.
        - Set resolved_at = NOW() for all bets.

    Create `src/jobs/footballRefreshOdds.ts`:
    - Export async function `runFootballRefreshOdds(job: Job)`:
      - **Purpose:** Refresh odds for fixtures within 24h of kickoff — odds may change close to match time.
      - Query football_matches WHERE status = 'NS' AND kickoff_at BETWEEN NOW() AND NOW() + INTERVAL '24 hours' AND (updated_at IS NULL OR updated_at < NOW() - INTERVAL '2 hours').
      - If no matches need refresh, return early.
      - For each match:
        - Fetch latest odds via apiClient.getFixtureOdds(match.fixtureId).
        - Parse odds and UPDATE football_matches SET home_odds, draw_odds, away_odds, exact_score_odds, updated_at = NOW().
        - If match has announcement_message_id, call updateLiveScoreEmbed() to update the embed with fresh odds.
      - Log summary: "Refreshed odds for N fixtures".

    Create `src/jobs/footballPollScores.ts`:
    - Export async function `runFootballPollScores(job: Job)`:
      - Query football_matches WHERE status IN ('1H', 'HT', '2H', 'ET', 'BT', 'P') AND kickoff_at > NOW() - INTERVAL '2 hours'.
      - If no active matches, return early.
      - Batch fixture IDs into groups of MAX_BATCH_FIXTURE_IDS (20).
      - For each batch: call apiClient.getFixtureScores(fixtureIds).
      - For each returned fixture:
        - UPDATE football_matches SET home_score = X, away_score = Y, status = new_status WHERE fixture_id = X.
        - Call updateLiveScoreEmbed(match, client) to update the Discord embed.
      - If any match status changed to FT/AET/PEN: trigger resolveMatchBets() for that match.
      - Log summary: "Polled N active matches, updated M scores".

    Create `src/jobs/footballResolveMatches.ts`:
    - Export async function `runFootballResolveMatches(job: Job)`:
      - Query football_matches WHERE status IN ('FT', 'AET', 'PEN') AND id IN (SELECT fixture_id FROM football_bets WHERE status = 'pending').
      - For each match:
        - If result not yet fetched (scores are null), call apiClient.getFixtureResult(fixtureId) to get final score.
        - UPDATE football_matches with final scores and status.
        - Call resolveMatchBets(match, db) to process all pending bets.
        - Send DM notification to winners (optional — use REST API).
      - Also check for matches that should have finished but are still in live status (kickoff_at + 2 hours elapsed, status still 2H):
        - Fetch latest status via apiClient.getFixtureResult(fixtureId).
        - If status changed to FT, process as above.
        - If status is PST/CANC/ABD, process as void refund.
      - Log summary: "Resolved N matches, processed M bets".
