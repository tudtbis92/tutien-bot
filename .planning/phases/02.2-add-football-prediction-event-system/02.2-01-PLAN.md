## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Discord component → bot handler | User clicks prediction button — input must be validated (fixture ID, bet amount) |
| API-Football → bot | External API responses must be validated before DB storage |
| User balance → bet deduction | Atomic transaction prevents double-spend |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.2-01 | Tampering | football_bets.wager_amount | mitigate | DB CHECK constraint `wager_amount >= 0` + Zod validation in component handler |
| T-02.2-02 | Tampering | Bet placement race condition | mitigate | Atomic `UPDATE users SET balance = balance - $amount WHERE id = $1 AND balance >= $amount` in transaction |
| T-02.2-03 | Spoofing | Component customId fixture_id | mitigate | Validate fixture_id exists in football_matches table before processing bet |
| T-02.2-04 | Information Disclosure | API-Football keys in logs | mitigate | Keys from env only, redact in error messages, never log key values |
| T-02.2-05 | Tampering | Prediction edit after kickoff | mitigate | Server-side kickoff_at check on every edit — reject if NOW() >= kickoff_at |
| T-02.2-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages required — all existing dependencies |
</threat_model>
