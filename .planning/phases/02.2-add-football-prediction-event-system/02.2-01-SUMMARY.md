## Summary of Changes
Implemented the baseline foundation layer for the Football Prediction Event System. This includes 4 Drizzle database schemas, 2 configuration/constant files, and registration in the primary schema index. The TypeScript project compiles with zero warnings or errors, and the Drizzle migration file has been generated and cleaned.

### 1. Database Schema Additions
- **`football_matches` (`src/db/schema/footballMatches.ts`)**
  - Maps to API-Football fixtures (`fixture_id` unique).
  - Tracks league details, teams, kickoff times (`kickoff_at` with index), status (`status` with index), scores, and decimal/correct score odds.
  - Integrates Discord metadata columns (`announcement_channel_id`, `announcement_message_id`) for live embed updates.
- **`football_bets` (`src/db/schema/footballBets.ts`)**
  - Stores user predictions for a match (either result-based or exact score-based).
  - Enforces atomic wagers (`wager_amount` as BigInt with a DB-level CHECK constraint `>= 0` to prevent negative balance overflow/tampering).
  - Constrains users to one bet per match per type using a unique index on `(user_id, fixture_id, bet_type)`.
  - References `users.id` and `football_matches.id` with proper foreign keys.
- **`api_cache` (`src/db/schema/apiCache.ts`)**
  - Implements lightweight persistent API cache with hash key (`cache_key` unique) and expiration tracking (`expires_at` with index) for rate limit handling.
- **`prediction_channels` (`src/db/schema/predictionChannels.ts`)**
  - Manages channel/league prediction notification settings.
  - Uses `(guild_id, channel_id, league_id)` unique composite index, where `league_id = 0` acts as a global toggle for the channel.

### 2. Constants & Configuration
- **`CURATED_LEAGUES` (`src/constants/footballLeagues.ts`)**
  - Exposes the 10 target leagues (Premier League, La Liga, Ligue 1, Bundesliga, Serie A, Champions League, Europa League, Conference League, World Cup, Euro) with their API-Football IDs.
  - Provides a helper `getLeagueIds()` returning a flat array of IDs.
- **`FOOTBALL_CONFIG` (`src/constants/footballConfig.ts`)**
  - Controls system boundaries: `MIN_BET = 100n`, `MAX_BET = 1,000,000n` (preventing economic inflation), prediction windows (open 24 hours prior), polling intervals (15 minutes), and API batch query sizes.

### 3. Migrations & Indexing
- **Schema Index (`src/db/schema/index.ts`)**
  - Registered all 4 new schema files so that Drizzle ORM and `drizzle-kit` can resolve them during build and migration.
- **Migration Generation (`migrations/0003_fat_pandemic.sql`)**

  - *Cleaned the migration file*: Removed duplicate statements for `gather_pool_items`, `recipes`, and `items` that were re-generated because of a missing local `0002_snapshot.json` snapshot file, ensuring migrations run successfully without failing on already-existing resources.

The migration SQL is complete and ready to be run once the connection string is provided.
