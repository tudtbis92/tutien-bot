## Accomplishments
- Created `buildPredictionEmbed` and `buildLiveScoreUpdate` with robust fallback translations, full matches odds visualization, dropdown (Win/Draw/Lose result selection), and buttons (8 popular score predictions) that direct user selections to wager modals.
- Created `predictionService.ts` containing the core logic for checking wagers boundaries, validating kickoffs, resolving specific odds, locking user balance via `SELECT FOR UPDATE`, and atomically refunding old bets when editing.
- Created `predictResult.ts` and `predictScore.ts` handlers to trigger interactive Discord input modals for bet wagers.
- Created `predictModal.ts` handler to Zod-validate modal inputs and process bets through `predictionService.ts`.
- Integrated all prediction select menus, buttons, and modals into the core `interactionCreate.ts` event gateway router.
- Developed a comprehensive unit test suite in `predictionService.test.ts` (9 tests) verifying all edge cases of placing, editing, refunding, and validating wagers using deterministic Drizzle ORM chain mocks. All 47 tests across the project pass flawlessly.

## Decisions Made
- Used stateless custom IDs (like `predict:modal:{betType}:{matchId}:{prediction}`) to capture all critical selection context directly within Discord components, avoiding any memory leaks or complex cache lookup tables.
- Implemented defensive database constraints in `predictionService.ts` to ensure users are completely blocked from placing bets after kickoff, and that balance deductions never allow negative values under any conditions.
