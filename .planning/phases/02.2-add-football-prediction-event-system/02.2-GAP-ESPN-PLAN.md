# Objective
Pivot the football prediction system from the restricted RapidAPI to a free Hybrid Model: **ESPN API (Fixtures/Scores)** + **Playwright (DraftKings Correct Score Odds)**.

# Background & Motivation
The previously selected API (`free-api-live-football-data`) has a severe limitation of 100 requests per month on the free tier, making it unsustainable for a bot that needs to poll live scores every 15 minutes. ESPN provides a completely free, undocumented API that covers all major leagues and provides basic odds (Moneyline, Over/Under, Spread) via DraftKings. However, it lacks "Correct Score" odds. To satisfy requirement PRED-09 (Correct Score predictions), we will use Playwright to crawl the DraftKings event page directly using the Event ID provided by ESPN.

# Scope & Impact
- Database schema update (`football_matches`) to add `dkEventId`.
- Rewriting the API client to fetch from ESPN's scoreboard endpoint.
- Updating background jobs to handle ESPN's polling and introduce a new crawling job.
- Creating a new Playwright crawling service to bypass DraftKings WAF and extract correct score odds.

# Proposed Solution

## 1. Schema & Constants Update
- **`src/db/schema/footballMatches.ts`**: Add a `dkEventId` (varchar) column to store the DraftKings ID extracted from ESPN.
- **`src/constants/footballLeagues.ts`**: Replace numeric API-Football IDs with ESPN slugs (e.g., `39` -> `'eng.1'`, `140` -> `'esp.1'`).

## 2. ESPN API Client Rewrite
- **`src/services/football/apiClient.ts`**: Completely rewrite `FootballApiClient` to fetch from `https://site.api.espn.com/apis/site/v2/sports/soccer/{league}/scoreboard`.
- **`src/services/football/oddsCalculator.ts`**: Add `convertAmericanToDecimal` function. Update logic to extract Moneyline odds and the DraftKings link from the ESPN JSON response.

## 3. Playwright DraftKings Crawler
- **`src/services/football/draftKingsCrawler.ts`**: Create a new service.
  - Launches an isolated Playwright Chromium instance.
  - Navigates to `https://sportsbook.draftkings.com/event/{dkEventId}`.
  - Waits for the "Game Props" or "Correct Score" section to render.
  - Extracts the American odds for scores (1-0, 2-1, etc.) and converts them to Decimal.
  - Returns a mapping of `score -> decimal_odds`.

## 4. Job Orchestration
- **`src/jobs/footballFetchFixtures.ts`**: Update to use ESPN client. Extracts and saves `dkEventId` alongside match data.
- **`src/jobs/footballCrawlOdds.ts`** (NEW): A dedicated pg-boss job that runs once daily (or 12h before kickoff). Queries matches with a valid `dkEventId` and missing `exactScoreOdds`. Runs the Playwright crawler and updates the database.
- **`src/jobs/footballPollScores.ts`**: Update parsing logic to match ESPN's status codes (`STATUS_IN_PROGRESS`, `STATUS_FINAL`) and scoring structure.
- **`src/jobs/footballResolveMatches.ts`**: Adjust logic if ESPN data format differs slightly for finalized matches.

# Alternatives Considered
- **Continuing with RapidAPI**: Rejected due to the 100 req/month limit, which would quickly be exhausted.
- **Dropping Correct Score**: Rejected as it is a core "high risk, high reward" feature of the bot's Tu Tiên economy.

# Implementation Steps
1. Update `footballMatches` schema and generate migration.
2. Update `footballLeagues.ts` with ESPN slugs.
3. Rewrite `apiClient.ts` and `oddsCalculator.ts` for ESPN data.
4. Implement `draftKingsCrawler.ts` using `@playwright/test`.
5. Refactor `footballFetchFixtures.ts` and `footballPollScores.ts`.
6. Create `footballCrawlOdds.ts` job and register it in `pgBoss.ts`.
